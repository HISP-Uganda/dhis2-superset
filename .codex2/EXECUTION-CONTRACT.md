# EXECUTION-CONTRACT.md
## Formal Delivery Contract for Codex

## 1. Contract Purpose
This contract defines the binding execution rules for implementing the required UI modernization, workflow cleanup, DHIS2 Database enhancement, theme system, public dashboard layout controls, and generic staging/storage architecture.

## 2. Binding Obligation
By proceeding, the implementation agent agrees to deliver a fully integrated, tested, documented, production-grade solution satisfying the project scope and all mandatory rules.

No milestone or workstream may be considered complete unless:
- implementation is complete
- tests for that scope pass
- documentation is updated
- working features remain intact

## 3. Mandatory Delivery Requirements
The agent MUST deliver:
1. professional UI modernization across the platform
2. improved application shell and navigation
3. cleaner page layouts and better information hierarchy
4. dynamic layout settings for public dashboards
5. comprehensive theme system with professional blended color/background behavior
6. updated dataset creation workflow centered on Database selection
7. DHIS2 Database model supporting multiple configured DHIS2 connections
8. generic local staging/storage for supported sources where enabled
9. staged-data-first chart and visualization querying
10. strong non-regression handling and passing tests
11. migrations
12. docs and runbooks

## 4. Quality Threshold
The implementation is only acceptable if it is:
- production-ready
- visually professional
- original in style
- maintainable
- extensible
- responsive
- test-covered
- non-regressive
- clearly documented

## 5. Mandatory Technical Constraints
The agent MUST NOT:
- break working features
- progress with failing tests
- ask for approval on already-scoped requirements
- copy proprietary design assets or branding
- leave workflows cluttered or conceptually duplicated
- leave staged chart execution dependent on live source queries by default
- disable mandatory staged background processing

## 6. Execution Standard
The agent MUST:
- make professional decisions autonomously
- preserve backward compatibility where required
- modernize carefully without introducing regressions
- use test-gated milestone delivery
- keep the result original, polished, and enterprise-grade
