# DELIVERY-RULES.md
## Delivery Rules for Codex Agent

1. Treat all `.codex/*` files as binding delivery authority.
2. Working features MUST NOT be broken.
3. Do not introduce regressions while modernizing the UI.
4. Do not skip migrations where data model changes are introduced.
5. Do not skip UI updates where backend changes affect workflows.
6. Do not treat styling-only changes as sufficient modernization.
7. Do not leave workflows cluttered or duplicated.
8. Do not ask for approval when the requirement is already in scope.
9. Do not progress to the next milestone until current milestone tests pass.
10. Use an original enterprise BI visual language; do not copy proprietary branding.
11. Keep colors, backgrounds, surfaces, typography, and spacing professionally blended.
12. Keep the platform responsive and performant.
13. Preserve observability and error clarity.
14. Keep staging architecture generic enough for supported staged sources.
15. Keep DHIS2 multi-connection behavior aligned to the Database concept in the UI.
