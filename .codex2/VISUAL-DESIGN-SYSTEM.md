# VISUAL-DESIGN-SYSTEM.md
## Visual Design and Layout System Requirements

## 1. Visual Direction
The design language must be:
- clean
- modern
- enterprise-grade
- professional
- minimal without feeling empty
- highly readable
- visually coherent

The target aesthetic should be comparable in polish to top analytics platforms, including Power BI-level enterprise maturity, but implemented as an original design system.

## 2. Mandatory Design Tokens
Define or refactor reusable tokens for:
- primary / secondary / accent colors
- neutral surfaces
- page backgrounds
- border colors
- semantic states
- typography scale
- spacing scale
- container widths
- radii
- shadows
- density modes

## 3. Blending Requirements
Colors, cards, backgrounds, nav surfaces, tables, forms, and dashboard containers must blend professionally.
Avoid harsh or disconnected color combinations.
Avoid muddy or low-contrast surfaces.
Avoid unbalanced dark/light mixes.
Surfaces should feel intentionally layered.

## 4. Component Style Requirements
Components must present a unified visual language for:
- nav items
- cards
- panels
- tables
- form sections
- tabs
- filters
- dashboard containers
- empty states
- loading states
- error states

## 5. Originality Requirement
Do not copy Microsoft's trademarks, logos, icons, or exact visual identity.
The goal is best-in-class enterprise BI polish, not imitation.
