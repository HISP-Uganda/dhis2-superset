# THEME-ARCHITECTURE.md
## Theme Architecture Requirements

## 1. Theme Layers
The implementation must support:
- internal platform theme
- dashboard presentation theme
- public dashboard theme

## 2. Theme Capabilities
Themes should control:
- color palette
- typography
- spacing density
- surfaces/backgrounds
- borders and shadows
- navigation appearance
- dashboard card style
- public dashboard presentation

## 3. Working Theme Presets
Support professional presets such as:
- Executive Light
- Executive Dark
- Clinical Blue
- Government Clean
- Minimal Analytics
- Brand Custom

## 4. Blending Rule
Theme combinations must remain visually coherent.
Backgrounds and accents must blend professionally and preserve readability.

## 5. Maintainability Rule
Themes must be implemented through structured settings/tokens, not page-by-page style hacks.
